# 캘린더 관련 구현에 대한 시나리오

- 호스트인 사용자의 username으로 캘린더 정보를 가져옵니다.
    - 캘린더 정보는 주제들, 설명을 반환합니다.
    - 자기 자신의 캘린더 정보를 가져오는 경우 주제들, 설명, 구글 캘린더 ID를 반환합니다.
- 존재하지 않는 사용자의 username으로 캘린더 정보를 가져오려고 하면 404 응답을 반환합니다.
- 호스트가 아닌 사용자의 username으로 캘린더 정보를 가져오려고 하면 404 응답을 반환합니다.
- 호스트 사용자는 유효한 캘린더 정보를 제출하여 캘린더를 생성할 수 있습니다.
    - 유효한 캘린더 정보는 주제들, 설명, 구글 캘린더 ID입니다.
- 캘린더가 있는 상황에서 추가 생성하려고 하면 422 응답을 반환합니다.
- 게스트 사용자가 캘린더를 생성하려고 하면 422 응답을 반환합니다.
- 로그인하지 않은 사용자가 캘린더를 생성하려고 하면 401 응답을 반환합니다.
- 사용자가 변경하는 항목만 변경되고 나머지는 기존 값을 유지합니다.
    - 변경할 수 있는 항목은 주제들, 설명, 구글 캘린더 ID입니다.
- 주제들은 사용자가 제출하는 순서를 유지합니다.
- 주제는 하나 이상 있어야 합니다.
- 구글 캘린더 ID는 항상 있어야 하며, 90자 이상입니다.
- 설명은 10글자 이상 입력해야 합니다.

- 호스트 사용자는 유효한 캘린더 정보를 제출하여 캘린더를 생성할 수 있습니다.
    - 유효한 캘린더 정보는 주제들, 설명, 구글 캘린더 ID입니다.
    - 대화 주제는 호스트가 입력한 순서를 유지합니다.
- 캘린더가 있는 상황에서 추가 생성하려고 하면 422 응답을 반환합니다.
- 게스트 사용자가 캘린더를 생성하려고 하면 403 응답을 반환합니다.
- 사용자가 변경하는 항목만 변경되고 나머지는 기존 값을 유지합니다.
    - 변경할 수 있는 항목은 주제들, 설명, 구글 캘린더 ID입니다.
    - 주제들은 사용자가 제출하는 순서를 유지합니다.
    - 주제는 하나 이상 있어야 합니다.
    - 구글 캘린더 ID는 항상 있어야 하며, 90자 이상입니다.
    - 설명은 10글자 이상 입력해야 합니다.
    - 캘린더가 없으면 404 응답을 합니다.

- 변경할 수 있는 항목은 주제들, 설명, 구글 캘린더 ID입니다.
- 주제들은 사용자가 제출하는 순서를 유지합니다.
